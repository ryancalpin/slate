Architecture Overview

Plugin-first design — core is thin, everything is a module. Does this structure look right?

System Architecture
CORE (thin)
├── Template Engine
│ └── loads/saves template JSON
├── Canvas System
│ ├── Grid mode (default)
│ ├── Freeform mode
│ └── Sections mode
├── Plugin Registry
│ └── discovers & registers modules
├── Storage Layer (Dexie/IndexedDB)
└── Export Engine
    ├── PDF (pixel-perfect)
    ├── PDF (clean doc)
    └── Print
MODULE PLUGIN INTERFACE
Every module exports:

├── meta — id, name, version, author
├── schema — config + data fields
├── defaultConfig — sane defaults
├── Renderer — view component
├── Editor — config/edit panel
├── PrintView — PDF/print output
└── minSize — grid units (w×h)

DATA FLOW
Template JSON (IndexedDB)
  ↓ loaded by Template Engine
Canvas renders modules via Plugin Registry
  ↓ user edits data in module Renderers
Changes saved back to IndexedDB (auto-save)
  ↓ export triggered
Export Engine → PDF (pixel/clean) or Print
